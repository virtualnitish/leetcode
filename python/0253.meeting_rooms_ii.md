# 253. Meeting Rooms II

## Line Sweep Method

```python
def minMeetingRooms(self, intervals: List[Interval]) -> int:
    # Extract start and end times separately
    starts = sorted([i.start for i in intervals])
    ends   = sorted([i.end for i in intervals])
    
    # Two pointers: s for starts, e for ends
    s = e = 0
    active = 0       # current number of ongoing meetings
    max_rooms = 0    # maximum concurrent meetings seen
    
    # Sweep through all start times
    while s < len(starts):
        # Case 1: a meeting starts before the earliest one ends
        if starts[s] < ends[e]:
            active += 1                  # need a new room
            max_rooms = max(max_rooms, active)
            s += 1                       # move to next start
        # Case 2: a meeting has ended before the next one starts
        else:
            active -= 1                  # free up a room
            e += 1                       # move to next end
    
    return max_rooms

```

### 🧩 Intuition
The problem is asking:  
**What is the maximum number of overlapping meetings at any point in time?**

That number = minimum number of rooms (or days) required.

Instead of juggling intervals directly, we separate **start times** and **end times**:
- Every start means “a new meeting begins → need one more room.”
- Every end means “a meeting finishes → free up a room.”

By sweeping through time in order, we can track how many meetings are active at once.

---

### ✅ Sweep-line Algorithm
1. Extract all start times and end times into two arrays.
2. Sort both arrays.
3. Use two pointers (`s` for starts, `e` for ends).
4. Walk through the starts:
   - If `start[s] < end[e]`:  
     → A new meeting starts before the earliest one ends → increment active count.  
     → Update max.
     → Move `s` forward.
   - Else (`start[s] >= end[e]`):  
     → A meeting has ended → decrement active count.  
     → Move `e` forward.
5. The maximum active count during this process = answer.

---

### ⚡ Example Walkthrough
Input:  
```
[(0,40), (5,10), (15,20)]
```

- Starts = [0, 5, 15]  
- Ends   = [10, 20, 40]

#### Step-by-step sweep:
- Start=0 < End=10 → active=1 → max=1  
- Start=5 < End=10 → active=2 → max=2  
- Start=15 ≥ End=10 → one meeting ends → active=1 → move end pointer  
- Now Start=15 < End=20 → active=2 → max=2  
- Next Start? none → done.

**Answer = 2**

---

### 🖼️ Visualization
Imagine a timeline:

```
0 ----------------------40
     5--10
          15---20
```

- At time 5, two meetings overlap (0–40 and 5–10).  
- At time 15, again two meetings overlap (0–40 and 15–20).  
- Peak overlap = 2 → need 2 rooms/days.

---

#### 🎯 Key Takeaway
- Sorting start and end times separately lets you “sweep” through events in chronological order.
- The maximum number of active meetings at any point = minimum rooms/days required.
- Complexity: (O(nlogn)) time (due to sorting), (O(n)) space.


## Heap Method
```python
import heapq

def minMeetingRooms(self, intervals: List[Interval]) -> int:
    if not intervals:
        return 0
    
    # Sort meetings by start time
    intervals.sort(key=lambda x: x.start)
    
    # Min-heap to track end times of ongoing meetings
    heap = []
    
    for meeting in intervals:
        # If earliest meeting ended before current starts → reuse room
        if heap and heap[0] <= meeting.start:
            heapq.heappop(heap)
        
        # Allocate room for current meeting
        heapq.heappush(heap, meeting.end)
    
    # Heap size = max rooms used concurrently
    return len(heap)

# Example usage:
meetings = [[0, 30], [5, 10], [15, 20]]
print(minMeetingRooms(meetings))  # Output: 2
```

### 🧩 Intuition
- We want to track ongoing meetings efficiently.
- A **min-heap** (priority queue) can store end times of meetings currently in progress.
- At each new meeting:
  - If the earliest meeting in the heap ends before the new one starts → pop it (reuse that room/day).
  - Otherwise → keep it (need a new room/day).
- Push the new meeting’s end time into the heap.
- The heap size at any point = number of rooms currently in use.
- The maximum heap size = answer.

---

### ⚡ Example Walkthrough
Input:
```
[(0,40), (5,10), (15,20)]
```

Sorted by start time:
```
[(0,40), (5,10), (15,20)]
```

#### Step-by-step:
1. Meeting (0,40): heap = [40] → 1 room.
2. Meeting (5,10): earliest end = 40 > 5 → overlap → push → heap = [10,40] → 2 rooms.
3. Meeting (15,20): earliest end = 10 ≤ 15 → pop → heap = [40].  
   Push 20 → heap = [20,40] → 2 rooms.

**Answer = 2**

---

### 🖼️ Visualization
Timeline view:

```
Room 1: (0----------------------------40)
Room 2:      (5--10)   (15---20)
```

At most 2 rooms are needed.

#### ✨ Key Notes for Interviews
- **Time complexity:** (O(n log n)) (sorting + heap operations).  
- **Space complexity:** (O(n)) (heap can hold all meetings in worst case).  
- Sweep-line and heap solutions are both optimal; heap is often easier to explain in terms of “rooms being allocated and freed.”
