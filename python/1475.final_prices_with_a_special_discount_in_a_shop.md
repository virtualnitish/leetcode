# 1475. Final Prices With a Special Discount in a Shop

```python
class Solution:
    def finalPrices(self, prices: List[int]) -> List[int]:
        n = len(prices)
        res = prices[:]   # start with original prices
        stack = []        # indices of items waiting for discount (monotonic increasing)
        
        for i in range(n):
            # If current price is <= last unresolved item,
            # it becomes the discount for that item
            while stack and prices[i] <= prices[stack[-1]]:
                past_item = stack.pop()
                res[past_item] -= prices[i]
            # push current index to maintain increasing order
            stack.append(i)
        
        return res
```


- **Problem goal**: For each item, subtract the price of the next item that is ≤ current price.  
- **Stack type**: Monotonic increasing (stores indices of higher prices).  
- **Condition**: `prices[i] <= prices[stack[-1]]` → today’s price is a valid discount for past item.  
- **Why increasing?** Because we’re searching for the *next smaller or equal element*. Storing higher prices ensures the first smaller/equal one resolves them immediately.  
- **Result meaning**: Each item’s final price after applying the discount.  
- **Complexity**: \(O(n)\) time, \(O(n)\) space.

### ASCII Stack Evolution Example  
For `prices = [8,4,6,2,3]`  

```
Day  Price  Stack (indices → prices)     Action
-------------------------------------------------
0    8      [0 → 8]                      push
1    4      [] → [1 → 4]                 pop 0 → res[0]=8-4=4, push
2    6      [1 → 4, 2 → 6]               push
3    2      [3 → 2]                      pop 2 → res[2]=6-2=4, 
                                         pop 1 → res[1]=4-2=2, push
4    3      [3 → 2, 4 → 3]               push
```

Final Result: `[4,2,4,2,3]`

---

### Mnemonic to Remember
- **Daily Temperatures** → monotonic **decreasing** stack (waiting for warmer days).  
- **Final Prices** → monotonic **increasing** stack (waiting for smaller/equal prices).
